1. 当前部署概览

本服务器承载面向 Codex / CPAM / OpenAI 兼容接口调用的代理链路。核心目标是：服务器位于中国大陆网络环境时，通过 Mihomo 订阅选择可用海外节点，再让 new-api、CPAM/CLIProxyAPI 或命令行工具按需走代理出口。

模块当前形态服务/容器名主要路径作用
Nginxsystemdnginx/etc/nginx/sites-available/api.aptim.topHTTPS 入口，反代 new-api

服务器上未发现 /data/cpam 目录；当前 CPAM 相关服务在运行形态上对应 /data/CLIProxyAPI 与 cli-proxy-api 容器。

2. 访问链路与协作关系

对外业务入口

用户 / 客户端
  -> https://api.aptim.top/
  -> Nginx 443 TLS
  -> http://127.0.0.1:3000
  -> new-api 容器
  -> CPAM / CLIProxyAPI 或上游模型服务
  -> Mihomo mixed / socks 入口
  -> 订阅节点出口

管理链路

管理员浏览器
  -> http://39.105.62.199:5178
  -> mihomo-manager-webui 容器
  -> SSH 管理通道 host.docker.internal:22
  -> 宿主机 systemd / /etc/mihomo / proxychains 配置

命令行代理链路

HTTP_PROXY / HTTPS_PROXY = http://127.0.0.1:7890
ALL_PROXY = socks5h://127.0.0.1:7891
proxychains4 <command> -> socks5 127.0.0.1 7891

5. 标准启动与重启顺序

systemctl restart mihomo
systemctl status mihomo --no-pager

# 2. CLIProxyAPI / CPAM
cd /data/CLIProxyAPI
docker compose up -d

# 3. new-api
cd /data/new-api
docker compose up -d

# 4. Mihomo Manager
cd /data/mihomo-manager-webui
docker compose -f docker-compose.yml -f docker-compose.ssh-key.yml up -d --build

# 5. Nginx
nginx -t
systemctl reload nginx

8. 备份与恢复

重点备份 new-api 数据、PostgreSQL 卷、CLIProxyAPI 账号认证、Mihomo 配置、Nginx 站点和 Mihomo Manager 的密钥配置。

BACKUP_DIR=/root/backups/mihomo-stack-$(date +%F-%H%M%S)
mkdir -p "$BACKUP_DIR"

tar -czf "$BACKUP_DIR/new-api-files.tgz" -C /data new-api/data new-api/logs
tar -czf "$BACKUP_DIR/cliproxyapi-files.tgz" -C /data CLIProxyAPI/config.yaml CLIProxyAPI/auths CLIProxyAPI/logs
tar -czf "$BACKUP_DIR/system-proxy-config.tgz" /etc/mihomo /etc/proxychains4.conf /etc/nginx/sites-available/api.aptim.top

docker run --rm \
  -v new-api_pg_data:/volume:ro \
  -v "$BACKUP_DIR":/backup \
  alpine sh -c 'cd /volume && tar -czf /backup/new-api-pg-volume.tgz .'

恢复原则：先恢复数据库和配置，启动 Mihomo，确认代理出口，再启动 CLIProxyAPI、new-api，最后检查 Nginx HTTPS。

10. 安全整改建议

收敛 new-api 直连端口

# /data/new-api/docker-compose.yml
ports:
  - "127.0.0.1:3000:3000"

cd /data/new-api
docker compose up -d
curl -I https://api.aptim.top/

保护敏感文件

chmod 600 /data/mihomo-manager-webui/secrets/*
chmod 600 /data/mihomo-manager-webui/.env
chmod 600 /data/CLIProxyAPI/config.yaml
chmod -R go-rwx /data/CLIProxyAPI/auths

不要把订阅链接 token、SSH 私钥、new-api 数据库密码、Redis 密码、CPAM 账号认证文件提交到 GitHub。
